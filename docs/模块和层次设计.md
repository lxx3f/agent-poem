# api模块
编写提供给前端交互的API，负责根据schema封装service层返回的数据

# LLM模块
提供LLM服务，包括构建pormpt

# schema
规定API层传入和传出的数据结构

# service
负责具体的业务处理逻辑，要求传入和传出的都是具体数据类型，不要过度封装

## mysql_service
提供数据库基本操作，不进行安全检查（交给具体的业务服务处理）

## user_service
提供用户相关业务操作，并进行安全检查

## potery_service
提供数据库诗词查询服务



# 业务逻辑
agent负责提供system prompt模版以及LLM配置，workflow负责具体处理方法，一个workflow对应处理一类任务，一个workflow可以被多个agent调用。

agent run的流程：
1. 将用户输入和配置信息传入workflow，启动workflow执行流程
2. workflow执行流程：
    - 根据用户输入调用不同的处理方法
    - 处理方法中调用LLM模块进行大模型推理
    - 处理方法中调用service模块进行具体业务处理
    - 返回最终结果给agent
3. agent将结果返回给用户
